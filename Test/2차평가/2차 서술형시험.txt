모범답안	
TCP : 연결기반으로 연결 후 통신하는 방식임.

- Byte-stream 으로 데이터 경계를 구분 안함.

- 신뢰성 있는 데이터 전송을 한다.

- 데이터의 수신여부를 확인함 (데이터가 손실되면 재전송됨)

- 패킷을 관리할 필요가 없다. (UDP 보다 속도가 느림)

- 데이터 전송 순서가 보장됨.

- 데이터의 전송속도보다 안정성이 중요한 경우 사용한다.

- TCP 패킷은 세그먼트

UDP : 비연결기반이며 연결없이 통신하는 방식임.

- datagram 으로 데이터의 경계를 구분함.

- 신뢰성 있는 통신을 하지 않는다.

- 데이터의 전송 순서가 바뀔 수 있음

- 데이터의 수신여부를 확인 안 함. (데이터가 손실되어도 할 수 없음)

- 패킷을 관리해 주어야 함. (TCP 보다 전송속도가 빠름)

- 실시간전송처럼 데이터의 전송속도가 중요한 경우 사용한다.

- UDP 패킷은 데이터 그램
