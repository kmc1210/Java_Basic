
	+ 배열, Array

	 - 같은 데이터타입의 여러 개의 변수를 하나의 묶음으로 사용하는 것
	 
	 - 여러 개의 변수를 한번에 선언할 수 있다.

	 - 선언하는 변수들의 이름을 고민하는 노력이 줄어든다.
	 
	 - 변수 선언코드를 여러 번 사용하지 않아도 된다.
	
	+ 배열의 선언
	 
	 - 배열 공간의 참조값을 저장할 수 있는 변수 선언

	  	데이터타입[] 배열명;	//추천
			

			ex)	int[] arr;
				double[] dArr;
				char[] chArr;
			


	  	데이터타입 배열명[];	//비추천

			ex)	int arr[];
				double dArr[];
				char chArr[];


	+ 배열의 생성 
	 
	 - 실제 데이터가 저장될 수 있는 공간(Element, 요소)을 생성한다.  

		new 데이터타입[개수];
		
		** 개수는 양수로 지정해야한다.

			ex)	int[] arr; //배열 변수 선언

				arr = new int[10]; //배열 생성


	+ 배열의 요소 접근(활용)
	 
	 - 배열의 인덱스(index)를 이용하여 요소(저장공간)에 접근할 수 있다.

	 - 인덱스는 0부터 시작하고 1씩 증가하면서 지정된다.

	 - 배열의 크기를 벗어난 인덱스 사용 불가. ex) 0~9까지의 배열일 경우 10을 쓰면 ArrayIndexOutOfBoundsException
		
	 - 음수 사용할 수 없다.
